Building a Recommender System with Flask

The DOrSU Program Recommender was born from a simple observation: incoming college students often struggle to choose which program fits them best. I wanted to build something that could guide that decision with data rather than guesswork.

How it works

The system uses a questionnaire-based approach. Students answer questions about their interests, academic strengths, and preferences. Each answer maps to weighted criteria across DOrSU's program offerings.

Here's a simplified version of the core matching logic:

def compute_match_scores(student_responses):
    scores = {}
    for program in programs:
        score = 0
        for criteria, weight in program.criteria.items():
            if student_responses.get(criteria) == weight.target:
                score += weight.value
        scores[program.name] = score
    return sorted(scores, key=scores.get, reverse=True)

Tech stack

Built with Flask on the backend, the app handles form submissions, computes match scores server-side, and renders results through Jinja2 templates. The front-end uses vanilla JavaScript for interactivity and a clean, responsive layout.

What I learned

This project taught me how to think about data-driven decision-making from a user's perspective. The hardest part wasn't the algorithm — it was designing questions that are clear, unbiased, and actually helpful. I also got more comfortable deploying Flask apps on Render and debugging in production.
