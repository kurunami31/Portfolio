Resibooth

Python Flask JavaScript HTML & CSS Render

Problem

Photo booths are popular at events, but physical booth hardware is expensive and location-bound. The idea behind Resibooth was to create a self-service photo booth platform that works entirely in the browser — no hardware needed.

Approach

The platform lets users capture photos via their device camera, apply customizations, and share their creations digitally. Key principles:

  • Zero downloads — everything works in the browser
  • Simple UX — capture, customize, and share in a few taps
  • Privacy-first — no uploads to external servers unless the user chooses to share

Implementation

The core functionality uses the browser's MediaDevices API for camera access and Canvas for photo capture and customization. Flask serves the application with a lightweight backend. The interface is designed to be intuitive — users should figure it out without instructions.

Result

A fully functional virtual photo booth that works on any device with a camera. Users can take photos, apply simple filters or frames, and download or share their creations.
